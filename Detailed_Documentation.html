Technical Manual v1.0

A comprehensive guide to the GamingX MERN stack ecosystem, covering development, management, and deployment.

1. System Architecture

The project follows a decoupled architecture using the MERN stack:

  • Frontend (React): Handles all UI logic, state management (cart, admin mode), and interactive animations.
  • Backend (Node/Express): Provides a RESTful API to manage products and newsletter subscriptions.
  • Database (MongoDB Atlas): A cloud-hosted NoSQL database for scalable data storage.

2. Admin Dashboard Features

The Admin Dashboard is the nerve center of the application, accessible via the "Admin" button in the navbar.

Core Capabilities:

  • Analytics: Live charts showing sales performance and active user count.
  • System Health: Real-time pulse monitoring of the MongoDB connection.
  • Product Management: CRUD (Create, Read, Update, Delete) operations for all gear.
  • Image Preview: Visual validation of product images before saving.

3. Project Timeline (History)

The History of GamingX is an interactive story section that fetches milestones from the database:

  • Dynamic Milestone Fetching: Pulls events (Title, Date, Description, Images) directly from MongoDB.
  • Categorized Events: Events are tagged as start, milestone, launch, or result.
  • Order Preservation: Automatically sorts events based on the order field.

4. API Reference

Product Routes

GET /api/products          - Fetch all products
POST /api/products         - Add a new product
DELETE /api/products/:id   - Remove a product

Timeline Routes

GET /api/timeline          - Fetch all history events
POST /api/timeline         - Add a history milestone

Subscription Routes

GET /api/subscribers       - List all email signups
POST /api/subscribe        - User newsletter signup

4. Database Configuration

Your database is hosted on MongoDB Atlas. To maintain access:

  • IP Whitelisting: If your network changes, you must add your new IP to the "Network Access" tab in the Atlas dashboard.
  • Environment Variables: The backend uses a .env file to store the MONGO_URI securely.

5. Production Deployment

To move this project to the public internet:

  • Frontend: Deploy the /frontend folder to Vercel or Netlify.
  • Backend: Deploy the /backend folder to Render or Heroku.
  • Sync: Update the frontend axios URLs from serveo.net to your permanent Render URL.
